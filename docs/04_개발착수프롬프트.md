# 개발 착수 프롬프트 (단계별)

> 필름사진 EXIF 수정 툴 · v1.0 · 작성일 2026-06-28
> 관련: `01_기획_기능_구조_디자인시스템.md` · `02_PRD.md` · `03_아키텍처설계.md`

---

## 사용 방법 (먼저 읽기)

1. AI 코딩 도구(예: Claude Code)를 **이 프로젝트 폴더**에서 연다. 그러면 AI가 위 설계 문서 3종을 읽을 수 있다.
2. 아래 프롬프트를 **한 단계씩, 순서대로** 복사해 넣는다. 한 번에 여러 단계를 주지 않는다.
3. 각 단계 끝의 **"완료 기준"이 충족됐는지 직접 확인**한 뒤 다음 단계로 넘어간다. 안 되면 "아직 X가 안 된다"고 말하고 같은 단계를 보완한다.
4. 매 단계마다 AI에게 **"무엇을 했고 어떻게 직접 확인하는지"**를 쉬운 말로 설명하게 한다(비개발자 기준).

> 💡 핵심 순서 원칙: 기능을 넓게 벌이기 전에 **사진 1장을 끝까지 안전 저장(M0)** 하는 가장 얇은 경로부터 관통한다. 가장 위험한 부분(ExifTool 번들·stay_open)을 맨 먼저 끝낸다.

## 공통 규칙 (매 단계에 적용 — 첫 단계에서 한 번 알려주고 유지)

```
[프로젝트 공통 규칙 — 모든 단계에서 지켜줘]

역할: 너는 이 데스크톱 앱을 함께 만드는 개발 파트너야. 나는 비개발자라
무엇을 왜 했는지, 어떻게 직접 확인하는지 쉬운 말로 설명해줘.

설계 문서: 이 폴더의 01_기획..., 02_PRD.md, 03_아키텍처설계.md 를 먼저 읽고
그 결정을 따라. 충돌하거나 모호하면 멈추고 나에게 물어봐.

기술 스택: Tauri v2 + React + TypeScript + Tailwind CSS, EXIF 엔진은
ExifTool 사이드카(-stay_open 상주 모드), 파일 작업은 전부 Rust 쪽에서.

작업 방식:
- 한 번에 한 단계만. 내가 준 단계의 범위를 넘지 마.
- 파일을 실제로 쓰는 로직은 항상 "원본 백업 → 쓰기 → 검증읽기" 순서를 지켜.
- 저장 전에는 사용자의 원본 사진 파일을 절대 건드리지 마(메모리에서만 편집).
- 끝나면: ① 무엇을 했는지 ② 어떻게 직접 확인하는지 ③ 다음에 뭘 할지 알려줘.
- 임의로 다음 단계까지 진행하지 마.
```

# M0 · 관통 슬라이스 (가장 먼저 — 위험 제거)

목표: **사진 1장의 촬영일을 바꿔 안전하게 저장**하는 가장 얇은 경로를 끝까지 뚫는다.

### 단계 1 · 프로젝트 뼈대 + 빈 3분할 화면

```
[단계 1] Tauri v2 + React + TypeScript + Tailwind 로 새 프로젝트를 만들어줘.

- 03_아키텍처설계.md 8장의 폴더 구조를 따라줘.
- 화면은 일단 비어 있어도 좋으니 3분할 레이아웃 뼈대만 잡아줘:
  왼쪽(썸네일 자리) · 가운데(미리보기 자리) · 오른쪽(편집 폼 자리) · 하단 상태바.
- 01_기획...md 4장의 다크 필름톤 색상으로 기본 테마만 입혀줘.
- 앱이 실행돼서 빈 3분할 화면이 뜨는 것까지 확인시켜줘.

완료 기준: `개발 모드 실행` 시 다크 테마의 빈 3분할 창이 뜬다.
```

### 단계 2 · ExifTool 사이드카 + 상주(stay_open) 서비스 + 태그 읽기

```
[단계 2] ExifTool을 앱에 사이드카로 동봉하고, 상주 모드로 연결해줘.

- ExifTool 바이너리를 tauri.conf.json의 externalBin에 등록하고
  capabilities에 shell:allow-execute 권한을 줘(03_아키텍처설계.md 8장 참고).
- Rust에 ExifToolService를 만들어 `-stay_open True -@ -` 로 프로세스를
  앱 시작 시 1회 띄우고, 종료 시 정리하게 해줘(03_..md 3장 결정1).
- 사진 1장 경로를 받아 기존 EXIF 태그를 JSON으로 읽어오는
  Tauri command(load_image)를 만들어줘.
- 화면에 사진 한 장을 골라 그 태그(특히 DateTimeOriginal)를
  텍스트로 출력해 읽기가 되는지 확인시켜줘.

완료 기준: 사진을 하나 골라 기존 촬영일 등 EXIF가 화면에 표시된다.
ExifTool은 매번 새로 뜨지 않고 상주 프로세스로 동작한다.
```

### 단계 3 · 1장 안전 저장 (백업 → 쓰기 → 검증읽기) ← **M0 완료점**

```
[단계 3] 사진 1장의 DateTimeOriginal을 바꿔 안전하게 저장해줘.

- 화면에서 날짜를 입력받아(YYYY:MM:DD HH:MM:SS) save_changes command 호출.
- 저장은 03_아키텍처설계.md 3장 결정3의 4단계를 지켜:
  ① 날짜 형식 검증 ② 원본 _original 백업 유지 ③ 바뀐 태그만 쓰기
  ④ 쓴 직후 다시 읽어 의도값과 일치 확인.
- 결과(성공/실패·검증값)를 화면에 보여줘.

완료 기준:
- 저장 후 다른 EXIF 뷰어(예: macOS 정보 창)에서 촬영일이 바뀐 게 보인다.
- 원본을 되돌릴 수 있는 _original 백업 파일이 남는다.
- 손대지 않은 다른 EXIF 값은 그대로다.
이 단계가 끝나면 M0 완료 — 가장 위험한 통합이 검증된 거야.
```

# M1 · MVP (핵심 시나리오 전체)

목표: 02_PRD.md 2장의 핵심 시나리오(폴더 드롭 → 롤 공통 적용 → 구간별 날짜 → 저장)를 앱에서 끝까지.

### 단계 4 · 다중 불러오기 + 썸네일 그리드 + 다중 선택

```
[단계 4] 여러 장을 다루게 확장하자.

- 폴더/파일 드래그앤드롭과 선택으로 여러 이미지를 불러와줘
  (JPEG·TIFF·PNG·DNG, 미지원은 무시하고 건수 안내) — PRD FR-1.
- 왼쪽에 썸네일 그리드를 가상 스크롤로 그려줘(수백 장도 부드럽게).
- 다중 선택(클릭/Cmd·Ctrl+클릭/Shift 범위/전체 선택)과
  하단 "선택 N / 전체 M" 표시 — FR-2.
- 썸네일 디코딩은 메타데이터 작업과 분리해 병렬로(03_..md 6장).

완료 기준: 폴더를 드롭하면 썸네일이 뜨고, 여러 장을 선택할 수 있다.
```

### 단계 5 · 미리보기 + 편집 폼 2구역 UI

```
[단계 5] 가운데 미리보기와 오른쪽 편집 폼을 만들어줘.

- 선택한 사진을 가운데에 크게 미리보기 — FR-3.
- 오른쪽 폼을 두 구역으로 분리(01_기획..md 4-1, 02_PRD 6-2):
  [롤 공통] 카메라 제조사/모델·렌즈 제조사/모델·필름·현상소 + "전체에 적용" 버튼
  [컷별-선택분] 촬영 날짜·시간 + "선택 항목에 적용" 버튼
- 아직 동작 연결은 안 해도 돼. UI 모양과 입력칸만 정확히.

완료 기준: 2구역으로 나뉜 입력 폼과 미리보기가 보인다.
```

### 단계 6 · 롤 공통 적용 (메모리)

```
[단계 6] "전체에 적용"을 동작시키자 — FR-4.

- 03_아키텍처설계.md 3장 결정2(메모리 우선)대로, 적용은 메모리의
  세션 데이터만 바꾸고 디스크는 건드리지 마.
- 빈 칸은 덮어쓰지 말고, 적용된 컷에는 상태 배지("공통 적용됨")를 표시해줘.
- 과거 입력값 기반 자동완성도 넣어줘.

완료 기준: 공통 정보를 입력해 "전체에 적용"하면 모든 컷에 반영되고
배지가 표시된다. (저장 전이라 파일은 아직 안 바뀜)
```

### 단계 7 · 컷별 날짜 적용 + 날짜 보조

```
[단계 7] "선택 항목에 적용"과 날짜 입력 보조를 만들자 — FR-5, FR-6.

- 다중 선택한 컷에만 날짜를 적용하고 롤 공통 값은 유지.
- 날짜 보조 3종:
  · 구간 적용(선택 묶음에 같은 날짜)
  · 시간 자동 증가(같은 날 연속 컷에 1초/지정 간격씩 더해 순서 정렬)
  · 느슨한 입력(연·월만 입력 시 일/시 기본값 보충)
- 잘못된 날짜는 입력 시점에 인라인 경고.

완료 기준: 일부만 선택해 날짜를 주면 그 컷만 바뀐다. 자동 증가 시
컷들의 촬영시각이 순서대로 증가한다.
```

### 단계 8 · 필름·현상소 UserComment 매핑

```
[단계 8] 필름·현상소를 표준 규칙으로 저장하게 해줘 — FR-10.

- 03_아키텍처설계.md 4장 규칙대로 `Film: ...; DevLab: ...` 형태로
  UserComment에 직렬화하고, 같은 내용을 XMP에도 병행 기록.
- 직렬화↔파싱 규칙을 Rust 한 곳(모듈)에 두고, 불러올 때 다시
  각 칸으로 복원되게 해줘.

완료 기준: 필름·현상소를 입력·저장 후 다시 불러오면 같은 값이
각 칸에 복원된다.
```

### 단계 9 · 배치 저장 파이프라인 + 진행률 + 부분 실패 ← **M1(MVP) 완료점**

```
[단계 9] 여러 장을 한 번에 안전 저장하게 완성하자 — FR-7·FR-8·FR-9.

- 저장 전 변경 요약(어떤 컷의 어떤 태그가 어떻게 바뀌는지·건수) 확인 단계.
- 컷마다 4단계 파이프라인(검증→백업→쓰기→검증읽기)을 독립 실행
  — 한 장 실패해도 나머지는 계속(03_..md 7장).
- 진행률을 이벤트로 화면에 표시하고 취소 가능하게.
- 끝나면 "N 성공 / M 실패" 요약과 실패분 재시도 버튼.

완료 기준(= MVP 합격선, 02_PRD 9장):
폴더 드롭 → 롤 공통 적용 → 구간별 날짜 → 저장까지 막힘없이 되고,
저장 결과가 다른 뷰어에서 의도대로 보이며, 모든 저장에 복구 가능한
백업이 남고, 200장 규모에서 손상·충돌이 없다.
```

# M2 · 편의 기능

[단계 10] 프리셋을 만들자 — FR-11.
자주 쓰는 (카메라+렌즈+필름+현상소) 조합을 이름 붙여 저장하고,

### 단계 11 · 롤 세션 자동복원 + Undo/Redo

```
[단계 11] 안전·편의 장치를 추가하자 — FR-12, FR-13.
- 편집 의도(세션)를 앱 데이터 폴더에 주기적 자동저장 →
  앱을 껐다 켜도 진행 중이던 롤 상태가 복원(파일은 저장 전이라 무손상).

[단계 12] CSV 입출력을 만들자 — FR-14.

현재 편집 내용을 CSV로 내보낼 수 있게 해줘.
완료 기준: CSV로 컷별 날짜를 한꺼번에 입력할 수 있다.

### 단계 13 · 스캐너 메타데이터 정리

```
[단계 13] 스캔본에 스캐너가 남긴 메타데이터를 선별 삭제하자 — FR-17.

- 삭제 대상은 스캐너/스캔SW 출처 태그만: Software·ProcessingSoftware·HostComputer·
  스캐너 MakerNotes·스캐너 Make/Model·스캐너가 쓴 IPTC/XMP. .xmp 사이드카가 있으면 함께.
- 전체 삭제(-all=)는 금지. 선별 목록만 지우고, 저장 전 "지워질 항목" 요약을 보여줘 확인받아.
- 스캔 일시는 지우지 말고 디지털화 날짜(CreateDate/DateTimeDigitized)로 보존해
  촬영일과 구분되게 해. 삭제도 기존 안전 저장 파이프라인(백업→쓰기→검증읽기)을 그대로 사용.

완료 기준: "스캐너 정보 지우기"를 켜고 저장하면 스캐너 태그만 사라지고,
촬영일·디지털화 날짜·사용자가 입력한 카메라/렌즈/필름 정보는 그대로 남는다.
```

### 단계 15 · 필름 데이터베이스 자동완성

```
